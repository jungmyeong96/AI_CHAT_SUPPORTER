# 시스템 공통 행동 지침 (System Common Rules)

본 프로젝트의 모든 에이전트(Architect, Builder, QA 등)는 다음의 '사전 검증 및 승인 프로세스'를 필수적으로 준수해야 합니다.

## 🚨 [행동 지침: MD 브리핑 및 Y/N 승인 프로세스]

모든 에이전트는 파일의 **수정(Write/Overwrite)**을 포함하는 작업을 수행할 때, 다음 단계를 반드시 순차적으로 이행하십시오:

1. **변경 내용 분석 및 MD 우선 브리핑:**
   - **MD 변경 사항:** 소스코드 반영 전, `docs/` 및 `agent_md` 파일의 변경 내용을 먼저 요약하여 제시하십시오.

2. **사용자 승인 요청 (Confirmation):**
   - 위 내용을 모두 제시한 후, 반드시 마지막에 아래 문구로 질문하십시오:
     > **"위 MD 및 소스코드 변경 내용을 승인하시겠습니까? (Y/N)"**

3. **조건부 실행 (Strict Execution):**
   - **'Y'를 입력받은 경우:** 1. 먼저 모든 MD 파일(docs 및 agent config)을 수정하여 설계 무결성을 확보하십시오.
     2. 그 후 소스코드 및 캐시 파일을 업데이트하십시오.
   - **'N'을 입력받은 경우:** - 모든 파일에 대한 쓰기 작업을 즉시 중단하고, 변경 사항을 적용하지 않습니다.
     - 사용자의 피드백을 기다리거나 수정안을 다시 제시하십시오.

## 🔄 프로세스 순서 준수 (The Protocol)
1. 사용자의 신규 기능/변경 요청 인입
2. [변경/수정] 모드 진입 (Architect 호출 필요 여부 판단)
3. MD 변경안 선제시 및 사용자 Y/N 대기
4. Y 입력 시: **MD 수정 완료 → 소스 수정 수행**
5. 완료 후: 반영된 MD와 소스 내용을 최종 보고

## 🛡️ 예외 케이스
- [최초생성] 명령이 있을 경우: 위 절차를 생략하고 즉시 전체 프로젝트 기반 설계를 수행합니다.
- 단순 읽기(Read) 및 분석 작업은 승인 없이 수행 가능합니다.
